Execution semantics

  • The browser never loops sends or implies synchronous completion.
  • Each group send is accepted first, then queued, and only later delivered, failed, or canceled.
  • One group send reaches every member of that existing group, so pacing stays capped by the platform.

Proposed contract notice

No v1 API operations exist yet. This panel validates orchestration and observability language for a future platform contract.

Responsible use

  • Named Lists can reference existing groups only.
  • Failure spikes auto-pause instead of retrying forever.
  • Audit events should record create, pause, resume, and abort.
